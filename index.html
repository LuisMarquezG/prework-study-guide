Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Section elements are semantic elements that define a section in a document that contains thematic content.
  • HTML uses the id attribute as a way to identify elements and select them.
  • The attribute class works in a similar way to the id.
  • The tag ul is for creating unordered lists.
  • Inside the ul element we need to wrap each item in a li element. Each one represents a new bullet point in our list.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CCS: For styling an element directly on the HTML file, inlines styles apply to an specific html tag.
  • Internal style sheet: In the HTML file using a style element in the head element, this contains rules only apply to that page, I can configure CSS classes and IDs.
  • External CSS style sheet: This is preferred in most cases because it allows to keep all CSS rules in a separate file, which makes design changes easier and using the principle called "Separation of Concerns".
  • Selectors are used to target the HTML elements on our web pages that we want to style.
  • Declarations, contains property (ie: color) and value (ie: blue) that I want to apply to that selector.
  • DRY: Don't repeat yourself, optimize reusing or sharing pieces of code to reduce the size and complexity of the code.
  • How to link a HTML file to a CSS file using the tag link wiy=th the attribute rel (relationship) and href (for relative path)
  • The class selector => .name-of-the-class
  • Properties:
    • width
    • height
    • background-color
    • color
    • text-align
    • display
    • margin-left
    • margin-right
    • padding-left
    • font-size
    • border
    • box-shadow
  • The colors could have semantic references such as blue or hexadecimal code (hex#)

Git

  • git status: Checks what branch we are currently on.
  • git clone + SSH value: Clones a repo from GitHub.
  • git revert + SHA: For a commit that you want to undo.
  • git checkout main: For moving into to the main branch
  • git checkout -b branch name/explicit name of the change: Creates a new branch and switches to it. ie: git checkout -b feature/new-title
  • git add -A: To indicate that I want to add ALL the files in the working branch.
  • git commit -m"message": To create a snapshot of he staged changes along a timeline of a Git projects history. We use the -m flag to associate a message with our commit. Commit messages should be short descriptive messages of the changes implemented.
  • git pull origin main: To check that our working branch is sync with the main branch on GitHub. It is a good habit to always pull the base branch into your feature branch before pushing any code.
  • git push origin name/explicit name: To push the changes from the working branch to the main branch.
